ATOS Standard 004 — Project Governance Standard (PGS)

Version: 1.0 (Draft)

Purpose

Defines how organizational work is planned, executed, prioritized, and completed.

⸻

Project Lifecycle

1. Proposal
2. Review
3. Approval
4. Planning
5. Execution
6. Review
7. Completion
8. Retrospective
9. Knowledge Promotion

⸻

Required Project Fields

* Project ID
* Purpose
* Owner
* Priority
* Status
* Dependencies
* Risks
* Success Metrics
* Completion Criteria

⸻

Priority Framework

Projects shall be classified as:

* Critical
* High
* Medium
* Low
* Deferred

Priority reflects strategic importance, not urgency alone.

⸻

Project Reviews

Projects shall undergo periodic review to evaluate:

* Progress
* Risks
* Alignment
* Resource allocation
* Success metrics

⸻

Success Criteria

Projects consistently:

* Finish with defined outcomes.
* Maintain accountability.
* Produce reusable learning.

⸻
