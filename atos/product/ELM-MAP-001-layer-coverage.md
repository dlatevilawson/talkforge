Quick map of what already exists vs what departments must deepen.

| Layer | Current strength | Gap |
|---|---|---|
| **1 Human** | Welcome asks name + “what brought you”; AMD-001 protects identity | Deeper psychology fields not yet structured in product specs |
| **2 Moment** | Prepare / event title; FLA event-first; landing lists life moments | Product still under-indexes non-interview moments in live paths |
| **3 Emotion** | Safety copy; dignity language; fear of performance addressed | Scenarios rarely **declare** target emotions explicitly |
| **4 Decision** | One-path welcome; waitlist CTA; reduced nav doors | Friction map (delay/avoid/begin) not yet documented per surface |
| **5 Transformation** | Session momentum Arrival→plan; “leave with momentum” | Not every feature defines Arrival/Departure; emotional shift not instrumented |

### Highest-leverage next applications

1. Require ELM five-layer block on every new scenario / feature brief.  
2. Declare emotions on voice tracks and landing moment examples.  
3. Define Departure emotional shift for waitlist join and first voice session (already partial).  
4. When PPS unlocks: measure “belief in ability to communicate” as Layer 5 signal.
